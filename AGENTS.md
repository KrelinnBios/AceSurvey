# AceSurvey 项目维护指南

本文适用于整个仓库，供维护者和自动化编码代理在修改页面、资源与文档时参考。

## 项目定位

AceSurvey 是一个面向无性恋社群的静态问卷与卡片生成工具。

- 项目没有构建步骤，页面由原生 HTML、CSS 和 JavaScript 组成。
- `index.html` 是问卷填写、草稿保存、卡片预览和图片导出的唯一入口。
- 网站可直接打开，也可部署到任意静态托管平台。
- 页面内容用于社群问卷填写、自我梳理和身份表达辅助，不构成医疗、心理、法律或其他专业建议。
- 项目不提供远程提交接口，访问者填写和导出的内容应继续留在本地处理。

## 修改原则

1. 使用 UTF-8 保存文本，避免因 PowerShell 默认编码造成中文乱码。
2. 保持项目为可独立部署的静态页面，不引入必须依赖后端或构建流程的功能。
3. 修改问卷题项、草稿保存、卡片预览或导出逻辑时，同步检查移动端浏览器和 QQ 内置浏览器体验。
4. 完整字体与图片导出体验依赖 Google Fonts 和 html2canvas CDN；验证导出时需要网络可用。
5. 不把问卷问题或导出结果描述为诊断、医学结论、专业评估或价值判断。
6. Git diff 应只包含本次任务所需内容，避免混入批量格式化或临时导出文件。

## README 约定

- 结构固定为：项目简介 → 功能概览 → 界面预览 → 使用方式 → 技术信息 → 内容边界 → 许可协议 → 反馈与贡献，与本作者其他仓库保持一致。
- 项目简介结尾补一句范围说明（如“项目只描述当前实际提供的功能……”），不要只留功能段落。
- 功能概览的每条格式固定为 `- 四字标签：描述。`，不加粗；标签需正好四字，不要三字或五字。

## 检查清单

提交前根据改动范围人工确认：

- `index.html` 可直接在浏览器打开。
- 问卷填写、刷新后的草稿恢复、重置和导出流程正常。
- 卡片预览和导出的 PNG 排版没有明显错位。
- 窄屏下输入、滚动、预览和长按保存体验可用。
- 外链字体或 html2canvas 加载失败时，页面仍保留基本可用性。
